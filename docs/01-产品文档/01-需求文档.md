# 家居布局画板需求文档

## 1. 文档说明

- 文档状态：当前实现基线
- 适用版本：`zhijian-hub@0.1.0`
- 最后核对：2026-07-18
- 维护范围：当前产品目标、功能边界和验收标准
- 关联代码：`src/components/features/room-canvas/`、`src/lib/hooks/use-room-canvas.ts`

本文档用于明确家居布局画板当前的产品目标、功能边界和验收标准。尚未进入代码的设想不作为当前需求。

## 2. 产品定位

家居布局画板是一个运行在浏览器中的轻量空间与储物位置规划工具。用户可以通过网格画布创建房间、家具和房间级储物设备，也可以在家具中建立逻辑储物设备，并使用详情、搜索、撤销/重做和本地保存完成基础整理。

当前产品支持房间、家具、储物设备和物品，不包含账号、多人协作或服务端数据同步。

## 3. 用户与使用场景

### 3.1 目标用户

- 希望快速记录家庭房间分布的个人用户。
- 需要用简单画板整理空间关系的用户。
- 希望数据在同一浏览器刷新后仍能保留的用户。

### 3.2 核心场景

1. 用户打开页面，在空白画布上绘制房间。
2. 用户移动或缩放房间，形成大致布局。
3. 用户为房间重命名，便于识别。
4. 用户通过房间列表或搜索快速定位目标房间。
5. 用户通过撤销/重做修正误操作。
6. 用户刷新或重新打开页面后继续编辑上次保存的布局。
7. 用户在选中房间的详情面板中进入家具绘制模式，并在房间内创建和管理家具。
8. 用户把储物设备直接绘制在房间中，或作为逻辑容器放入家具，并可在两类位置之间转移。

## 4. 功能需求

### 4.1 画布与网格

- 页面必须提供占满可用视口的房间布局画布。
- 画布必须显示点阵网格，默认网格间距为 20 像素。
- 房间的创建、移动和尺寸调整默认吸附到网格。
- 当画布中没有房间、当前处于房间工具且没有进行交互时，必须显示绘制提示。

### 4.2 房间创建

- 默认激活房间工具，用户可在画布空白处按下并拖拽创建矩形房间。
- 房间宽度和高度达到最小尺寸后才允许创建，当前最小尺寸为 20 像素。
- 顶部工具栏必须提供“添加房间”按钮，在当前视口中心创建默认房间。
- 默认房间尺寸为 6 × 4 个网格。
- 新房间必须获得稳定且唯一的 ID，并按递增序号生成默认名称，例如“房间 1”。
- 创建完成后，新房间必须进入选中状态。

### 4.3 房间选择、移动与调整大小

- 点击或聚焦房间时必须选中该房间。
- 选中房间必须显示明确的选中边框和八方向缩放手柄。
- 用户可拖拽房间移动位置。
- 用户可拖拽八方向手柄调整房间尺寸。
- 房间尺寸不得小于最小尺寸。
- 方向键每次按一个网格移动选中房间。
- `Shift + 方向键` 每次按一个网格调整房间宽度或高度。

### 4.4 家具创建与编辑

- 选中房间后，右侧详情面板必须提供“添加家具”操作。
- 添加家具进入一次性绘制模式，只允许在目标房间内部拖拽创建，按 `Escape` 可取消。
- 家具默认按递增序号命名，例如“家具 1”，创建后自动选中并退出绘制模式。
- 家具坐标相对于所属房间保存；房间移动时家具必须自然跟随。
- 家具支持指针和键盘移动、八方向缩放、重命名、删除及转移所属房间。
- 家具位置和尺寸始终限制在所属房间内，房间不能缩小到无法容纳现有家具或房间级储物设备。
- 选中房间、家具或储物设备后必须打开右侧详情面板，展示名称、层级、尺寸或位置和当前可执行操作。
- 删除包含家具或储物设备的房间前必须显示影响数量，确认后级联删除，并允许通过一次撤销完整恢复。

### 4.5 储物设备创建与编辑

- 选中房间后，详情面板必须提供“添加储物设备”操作，并进入一次性绘制模式。
- 房间级储物设备必须包含相对房间的矩形，支持指针和键盘移动、八方向缩放、重命名、删除及右键菜单。
- 房间级储物设备的位置和尺寸始终限制在所属房间内，默认按递增序号命名。
- 选中家具后，详情面板必须可直接创建不含二维坐标的逻辑储物设备。
- 储物设备只能放在房间或家具中；详情面板必须提供符合层级规则的位置选择器。
- 储物设备移入家具时必须移除矩形；从家具移入房间时必须在房间中心生成合法默认矩形。
- 储物设备的位置切换、重命名、删除及父实体级联删除必须进入历史，并支持刷新恢复与搜索定位。

### 4.6 物品管理

- 房间、家具和储物设备详情必须提供“添加物品”操作，并把当前实体作为默认存放位置。
- 添加物品必须填写名称、正整数数量和存放位置；物品可直接放在房间、家具或储物设备中。
- 物品支持修改名称、数量和存放位置，也支持从当前位置详情中删除。
- 房间、家具和储物设备详情必须展示直接存放的物品及数量。
- 物品的添加、修改、移动和删除必须进入完整画板历史，并支持本地持久化与刷新恢复。

### 4.7 实体名称和删除

- 用户可通过房间、家具或房间级储物设备右键菜单、`Enter` 或 `F2` 打开对应重命名弹窗。
- 重命名输入框必须显示当前名称，并支持确认或取消。
- 用户可通过右键菜单、`Delete` 或 `Backspace` 删除获得焦点的空间实体；逻辑储物设备可从详情面板重命名和删除。
- 顶部工具栏和画布右键菜单必须提供清空画板功能。
- 画布为空时，清空操作必须处于禁用状态。

### 4.8 画布平移与缩放

- 顶部工具栏必须提供选取工具；激活后，拖拽画布空白区域可以平移视口。
- 画布缩放范围为 50% 至 200%，每次调整 10%。
- 用户可通过滚轮或右下角缩放按钮调整比例。
- 缩放必须以画布视口中心为锚点，避免中心内容明显偏移。
- 到达缩放上下限后，对应缩放按钮必须禁用。

### 4.9 撤销与重做

- 顶部工具栏必须提供撤销和重做按钮，并根据历史状态自动禁用。
- 系统最多保存 50 条完整画板内容快照。
- 房间、家具、储物设备和物品的创建、修改、移动、删除和清空必须进入历史记录。
- 一次连续拖拽或缩放只记录为一次历史操作。
- 工具切换、房间选择、搜索定位、画布平移和画布缩放不进入历史记录。
- 必须支持以下快捷键：
    - `Ctrl/Cmd + Z`：撤销。
    - `Ctrl/Cmd + Shift + Z`：重做。
    - `Ctrl + Y`：重做。

### 4.10 搜索、房间总览与定位

- 画布右上角必须提供搜索入口和全部房间入口。
- 搜索必须按房间、家具、储物设备或物品名称进行包含匹配，并忽略大小写。
- 搜索结果必须区分实体类型；家具、储物设备和物品结果显示当前完整所属路径。
- 全部房间面板必须显示当前所有房间。
- 点击结果后必须定位目标所属房间，选中并短暂高亮目标；逻辑储物设备和物品在详情中高亮。
- 定位时优先保持当前缩放比例；只有目标无法完整展示时才缩小画布。
- 搜索面板打开后必须自动聚焦搜索输入框。
- 结果列表必须支持上下方向键循环导航，搜索输入框按 `Enter` 可定位第一个结果。
- 按 `Escape` 必须关闭面板并恢复触发按钮焦点；点击面板外区域必须关闭面板。

### 4.11 右键菜单与浮层

- 画布右键菜单提供添加房间和清空画板操作。
- 房间右键菜单提供添加家具、添加储物设备、添加物品、修改名称和删除房间操作。
- 家具右键菜单提供添加储物设备、添加物品、修改名称和删除家具操作。
- 房间级储物设备右键菜单提供添加物品、修改名称和删除储物设备操作。
- 右键菜单必须根据视口边界调整位置，避免内容溢出屏幕。
- 点击菜单外任意区域或按 `Escape` 必须关闭菜单。
- 工具栏、搜索面板、缩放控件等画布浮层不得误触发房间绘制或画布拖拽。

### 4.12 本地持久化

- 房间、家具、储物设备和物品数据必须保存到浏览器 `localStorage`。
- 存储文档使用 `version: 2`，包含各实体计数器和房间、家具、储物设备、物品数组。
- 页面初始化时必须读取并校验存储数据。
- 不兼容、结构错误或无法解析的数据必须被忽略，页面仍可正常使用。
- 画板内容或命名计数器变化后必须延迟保存，当前延迟为 300 毫秒。
- 页面触发 `pagehide` 时必须立即保存最新数据。
- 清空画板后必须持久化空房间列表。
- 当前四类实体及其计数器持久化；不保存选择、工具、视口、缩放比例或撤销/重做历史。
- 旧 `version: 1` 房间文档必须在内存中迁移，保存时统一写入 `version: 2`。

### 4.13 可访问性与输入方式

- 画布、工具栏、菜单、弹窗和搜索区域必须提供可理解的语义角色和无障碍名称。
- 房间、家具和房间级储物设备必须可通过键盘聚焦和操作，逻辑储物设备必须可通过详情列表访问。
- 弹窗和菜单打开后必须管理焦点，关闭后恢复原触发元素焦点。
- 输入框必须关联可识别的标签。
- 触屏或粗指针设备上的主要操作按钮点击区域不得小于 44 × 44 像素。
- 系统必须响应 `prefers-reduced-motion`，减少非必要动画。

## 5. 视觉与体验要求

- 整体采用简洁、温和的中式简约风格，以朱砂红为主要强调色。
- 家具使用温润木色块面，储物设备使用淡朱砂收纳盒样式，两类实体必须易于区分。
- 房间与绘制预览使用 4 像素小圆角。
- 按钮与输入框使用 6 像素圆角。
- 工具栏、菜单和弹窗等浮层使用 10 像素圆角。
- 画布、网格和分隔线保持直角。
- Hover 状态使用背景色或文字色表达，不重复绘制与容器边框相同的外框。
- 动画应克制，主要使用 `transform` 和 `opacity`。

## 6. 当前数据要求

每个房间包含以下字段：

| 字段     | 类型     | 说明                        |
| -------- | -------- | --------------------------- |
| `id`     | `string` | 房间唯一标识                |
| `name`   | `string` | 房间名称                    |
| `x`      | `number` | 画板坐标系中的左上角 X 坐标 |
| `y`      | `number` | 画板坐标系中的左上角 Y 坐标 |
| `width`  | `number` | 房间宽度                    |
| `height` | `number` | 房间高度                    |

每件家具包含以下字段：

| 字段     | 类型     | 说明                        |
| -------- | -------- | --------------------------- |
| `id`     | `string` | 家具唯一标识                |
| `roomId` | `string` | 所属房间 ID                 |
| `name`   | `string` | 家具名称                    |
| `x`      | `number` | 相对所属房间左上角的 X 坐标 |
| `y`      | `number` | 相对所属房间左上角的 Y 坐标 |
| `width`  | `number` | 家具宽度                    |
| `height` | `number` | 家具高度                    |

每个储物设备包含 `id`、`name` 和以下两类互斥位置之一：

| 位置类型 | 必需字段                     | 说明                               |
| -------- | ---------------------------- | ---------------------------------- |
| 房间     | `roomId`、`rect`             | `rect` 为相对房间的坐标和尺寸      |
| 家具     | `furnitureId`，不包含 `rect` | 只记录逻辑父级，不记录家具内部坐标 |

## 7. 当前非功能需求

- 所有业务代码使用 TypeScript 严格模式。
- 指针移动期间使用 `requestAnimationFrame` 合并高频更新。
- 页面刷新后不应丢失已经成功写入本地存储的实体数据。
- 核心功能必须具有 Vitest 和 React Testing Library 自动化测试。
- 提交前必须通过 `npm run check`。

## 8. 当前范围外

以下能力当前没有实现，不属于本版本验收范围：

- 用户账号、权限与多人协作。
- 后端 API、数据库和云端同步。
- 布局导入、导出、分享和打印。
- 多画板或多项目管理。
- 旋转、复制、组合、对齐和图层管理。

## 9. 文档同步要求

后续变更必须在同一次提交中同步更新相关文档：

- 新增、删除或调整产品行为：更新本文档和《功能文档》。
- 调整模块职责、数据模型、状态流转、存储方式或依赖：更新《架构设计文档》；如果影响用户行为，同时更新本文档和《功能文档》。
- 只修改内部实现且不改变架构边界或用户行为：无需机械修改需求和功能描述，但必须确认架构文档仍与代码一致。
- 文档不得提前描述尚未落地的功能；规划内容应单独放入方案文档。
