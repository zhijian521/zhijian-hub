# 家庭储物系统分阶段实施计划

## 文档说明

- 文档状态：设计方案
- 适用版本：`zhijian-hub@0.1.0`
- 最后核对：2026-07-18
- 维护范围：家庭储物系统的开发顺序、阶段交付物、测试重点和下一步
- 关联代码：`src/components/features/room-canvas/`、`src/lib/`

上级方案：[家庭储物系统方案总览](./00-方案总览.md)

## 1. 实施原则

- 一次只引入一个清晰能力，保持每个阶段可运行、可测试、可回退。
- 当前需求、功能和架构文档只记录已经落地的内容。
- 每个阶段完成时同步更新方案状态和受影响的当前基线文档。
- 不提前创建后续阶段的空组件、空类型或未使用目录。
- 新实体必须同时接入历史、持久化、搜索和测试，避免只完成视觉展示。

## 3. 第一阶段：数据基础

### 3.1 目标

在不改变当前用户行为的前提下，为后续实体建立稳定的数据和迁移基础。

### 3.2 交付内容

- 建立 `CanvasDocumentV2`。
- 实现 version 1 到 version 2 的安全迁移。
- 把房间历史快照改为画板内容快照。
- 抽出文档状态、历史和持久化职责。
- 保持当前房间创建、编辑、搜索和定位行为不变。
- 补充迁移、历史和持久化测试。

### 3.3 完成标准

- version 1 房间数据加载后内容和默认名称序号不变。
- 保存后统一写入 version 2。
- 异常数据不会覆盖已有存储。
- 房间撤销/重做行为不变。
- 刷新后房间数据正常恢复。
- `npm run check` 通过。

### 3.4 实施结果

- `CanvasDocumentV2` 已包含四类扁平实体数组和独立计数器。
- v1 文档读取时在内存迁移，后续保存统一写入 v2；无效数据不会被初始化流程自动覆盖。
- `useCanvasDocument` 已统一管理文档 State/Ref、完整内容历史、计数器和持久化。
- `useRoomCanvas` 保留房间交互与视口职责，对组件暴露的接口和现有操作行为不变。
- 已补充存储迁移、实体关系、完整内容撤销/重做和旧序号延续测试。
- 格式、Lint、类型、自动化测试和真实浏览器回归均通过。

## 4. 第二阶段：家具

### 4.1 交付内容

- 在房间内创建家具。
- 移动、缩放、重命名和删除家具。
- 家具坐标限制在所属房间内。
- 建立右侧详情面板。
- 家具接入撤销/重做、持久化和搜索。

### 4.2 完成标准

- 不能在房间外创建家具。
- 移动和缩放不会超出所属房间。
- 删除房间时家具被正确统计和级联处理。
- 搜索家具后可以定位所属房间并高亮家具。

### 4.3 实施结果

- 房间详情支持进入一次性家具绘制模式，家具坐标和尺寸始终限制在所属房间内。
- 家具支持指针与键盘移动、八方向缩放、重命名、删除和转移所属房间。
- 右侧功能区已扩展为搜索、房间总览和当前位置详情，房间详情展示子家具列表。
- 房间缩小时会保留容纳现有家具所需的最小边界；移动房间时家具通过相对坐标自然跟随。
- 删除包含家具的房间会展示影响数量并级联删除，整次操作可一次撤销。
- 家具已接入完整内容历史、version 2 持久化、刷新恢复和搜索定位。
- 自动化测试与真实浏览器验证覆盖家具创建、边界、编辑、转移、级联删除和定位。

## 5. 第三阶段：储物设备

### 5.1 交付内容

- 创建房间级可视化储物设备。
- 创建家具内逻辑储物设备。
- 支持在房间和家具之间移动。
- 完成储物设备的重命名、删除和级联规则。
- 接入历史、持久化和搜索。

### 5.2 完成标准

- 储物设备只能放在房间或家具中。
- 房间级设备包含矩形，家具内设备不保存无意义坐标。
- 切换位置后画布和层级面板立即同步。

### 5.3 实施结果

- 房间详情支持进入一次性储物设备绘制模式，房间级设备复用家具的移动、缩放、键盘和空间边界规则。
- 家具详情支持直接创建不含二维坐标的逻辑储物设备，房间和家具详情分别展示直接子设备。
- 储物设备详情提供按房间分组的位置选择器；移入家具时移除矩形，移入房间时在中心生成合法默认矩形。
- 储物设备支持重命名、删除、右键菜单和父实体级联删除，完整操作接入一次撤销/重做。
- 搜索已覆盖储物设备并实时解析房间或家具路径，点击结果可定位所属房间并高亮画布节点或详情。
- 储物设备默认序号、实体关系和完整数据已接入 version 2 持久化与刷新恢复。
- 自动化测试覆盖两类创建、边界、位置互转、搜索、持久化、级联删除和撤销恢复。

## 6. 第四阶段：物品

### 6.1 交付内容

- 新增快速添加物品表单。
- 支持名称、数量和位置。
- 支持物品修改、移动和删除。
- 房间、家具和储物设备显示物品数量。
- 根据当前选择提供默认位置。

### 6.2 完成标准

- 物品只能放在房间、家具或储物设备中。
- 每件物品同一时间只有一个位置。
- 移动物品后完整位置路径立即更新。
- 刷新和撤销/重做不会破坏实体关系。

### 6.3 实施结果

- 房间、家具和储物设备详情均提供物品添加入口，当前实体自动成为默认位置。
- 物品表单统一支持名称、正整数数量和房间、家具、储物设备三类位置。
- 各容器详情展示直接物品与数量，物品可编辑、移动和删除。
- 物品变更接入完整内容历史、独立计数器、version 2 持久化和刷新恢复。
- 自动化测试覆盖添加、搜索、位置移动、删除、撤销和刷新恢复。

## 7. 第五阶段：全局搜索

### 7.1 交付内容

- 搜索房间、家具、储物设备和物品。
- 展示实体类型和完整位置路径。
- 点击结果后定位、展开并高亮目标。
- 保持完整键盘操作和焦点恢复。

### 7.2 完成标准

- 重名实体可以通过路径区分。
- 位置变化后搜索路径立即更新。
- 物品搜索结果可以定位到最终容器和详情项。

### 7.3 实施结果

- 搜索已覆盖四类实体，物品结果实时展示完整存放路径。
- 点击物品结果会定位所属房间、选中最终容器并高亮物品详情项。
- 搜索结果继续支持方向键循环、回车定位和 Escape 焦点恢复。

## 8. 第六阶段：可靠性

### 8.1 交付内容

- JSON 导入和导出。
- 删除影响统计与确认。
- 数据迁移与异常恢复测试。
- 完整键盘、无障碍和真实浏览器回归。

### 8.2 完成标准

- 用户可以导出全部家庭储物数据。
- 导入前校验文档版本和结构。
- 删除父实体前显示受影响的设备和物品数量。
- 级联删除可以作为一次操作撤销。

## 9. 全阶段测试重点

- 每种实体只能选择符合规则的父级位置。
- 家具坐标和尺寸不会超出所属房间。
- 位置变化后搜索路径立即更新。
- 家具转移房间后，其后代位置路径正确。
- 储物设备在房间和家具之间移动时正确增加或移除矩形数据。
- 删除容器时影响统计和级联结果正确。
- 撤销/重做可以完整恢复级联变化。
- version 1 数据迁移后房间内容和序号不变。
- 页面刷新后实体关系不丢失。
- 键盘导航、弹窗、菜单和详情面板不会误触发画布操作。

## 10. 下一步

下一次开发进入第六阶段，优先处理数据可靠性：

1. 增加完整画板 JSON 导出。
2. 增加带版本与关系校验的 JSON 导入。
3. 补齐导入覆盖确认和失败反馈。
4. 扩展父实体删除的物品影响回归测试。
5. 完成完整键盘、无障碍和真实浏览器回归。

第六阶段不引入账号、云同步或多人协作，继续保持单机 localStorage 架构。
