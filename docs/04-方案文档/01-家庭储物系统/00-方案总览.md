## 1. 产品目标

项目最终定位为一个以“空间位置、容器层级和全局搜索”为核心的可视化家庭储物快速查找系统。

系统需要解决两个核心问题：

1. 新买或新整理的物品能够快速放入正确位置。
2. 需要使用物品时，能够快速知道它位于哪个房间、家具或储物设备中。

当前房间画板继续作为空间基础，不推倒重做。后续在现有创建房间、视口定位、搜索、撤销/重做和本地持久化能力上逐步扩展。

## 2. 空间与储物层级

```text
家庭
└── 房间
    ├── 家具
    │   ├── 物品
    │   └── 储物设备
    │       └── 物品
    ├── 储物设备
    │   └── 物品
    └── 物品
```

### 2.1 层级规则

- 家具只能放在房间里。
- 储物设备可以放在房间或家具里。
- 物品可以放在房间、家具或储物设备里。
- 家具之间不嵌套。
- 储物设备之间暂不嵌套。
- 每个物品同一时间只能有一个位置。
- 同一层级允许重名，通过完整位置路径区分。

### 2.2 名词定义

| 实体                     | 说明                                             | 示例                   |
| ------------------------ | ------------------------------------------------ | ---------------------- |
| 房间 `Room`              | 家庭中的一级空间                                 | 客厅、主卧、厨房       |
| 家具 `Furniture`         | 固定放在房间中的空间对象，可直接存放物品         | 衣柜、电视柜、书桌     |
| 储物设备 `StorageDevice` | 可直接放在房间中，也可放入家具中的储物容器或设备 | 收纳箱、抽屉盒、医药箱 |
| 物品 `Item`              | 用户需要保存和快速查找的实际对象                 | 遥控器、电池、保鲜袋   |

## 3. 核心设计决策

### 3.1 画布负责大位置

画布展示房间、家具和直接放在房间中的储物设备。家具或储物设备可以显示物品数量，但物品默认不占用画布矩形。

### 3.2 层级面板负责容器内部

选择房间、家具或储物设备后，通过右侧详情面板查看和管理内部内容，避免物品数量增多后让画布失去可读性。

### 3.3 搜索连接空间和层级

搜索结果展示完整位置路径。点击结果后，系统先定位所属房间，再展开容器层级，最后高亮目标实体。

```text
保鲜袋
厨房 / 橱柜 / 左侧收纳盒
```

### 3.4 数据保持扁平

房间、家具、储物设备和物品分别保存在扁平数组中，通过 ID 建立父子关系。完整位置路径、物品数量和搜索结果均实时派生，不重复持久化。

### 3.5 先完成核心闭环

第一版只解决“快速放入、快速查找”，不提前加入图片识别、价格、购买渠道、保质期、复杂统计或多人协作。

## 6. 第一版完成标准

1. 可以创建房间和家具。
2. 可以把储物设备放入房间或家具。
3. 可以把物品放入房间、家具或储物设备。
4. 新物品可以根据当前选择快速确定位置。
5. 搜索物品后可以看到完整位置路径。
6. 点击搜索结果可以定位到对应房间和容器。
7. 刷新后所有实体数据仍然存在。
8. 所有实体变更支持撤销和重做。
9. version 1 房间数据可以安全迁移。
10. 可以导出数据进行备份。

## 7. 当前暂不实施

- 用户账号和多人协作。
- 云端数据库和跨设备同步。
- 图片识别、条码扫描和 AI 自动分类。
- 价格、购买渠道、保质期和复杂统计。
- 家具内部的精确二维布局。
- 储物设备之间的多层嵌套。
- 独立搜索服务或搜索引擎。

这些能力只有在核心流程稳定，并出现明确使用需求后再进入新方案。
