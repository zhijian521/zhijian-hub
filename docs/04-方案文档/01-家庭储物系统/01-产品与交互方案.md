# 家庭储物系统产品与交互方案

## 文档说明

- 文档状态：设计方案
- 适用版本：`zhijian-hub@0.1.0`
- 最后核对：2026-07-18
- 维护范围：家具、储物设备、物品、详情面板和全局搜索的用户交互
- 关联代码：`src/components/features/room-canvas/`、`src/components/ui/`

上级方案：[家庭储物系统方案总览](./00-方案总览.md)

## 1. 页面结构

系统采用“空间画布 + 右侧层级面板 + 全局搜索”的组合。

### 1.1 空间画布

画布继续展示：

- 房间。
- 房间中的家具。
- 直接放在房间中的储物设备。
- 家具或储物设备上的物品数量提示。

家具坐标相对于所属房间计算。移动房间时，所属家具和房间级储物设备在视觉上自然跟随。

物品默认不绘制为画布矩形，只有确认存在精确空间规划需求时才增加物品坐标。

### 1.2 右侧详情面板

当前右上角搜索与房间列表功能区逐步扩展为右侧详情面板。面板负责：

- 展示当前位置路径。
- 展示当前实体信息。
- 展示直接子实体和物品。
- 提供当前层级允许执行的新增操作。
- 提供重命名、移动位置和删除操作。

```text
客厅
├── 电视柜
│   ├── 遥控器
│   └── 左侧收纳盒
│       ├── 电池
│       └── 数据线
├── 医药箱
│   └── 创可贴
└── 备用灯泡
```

## 2. 上下文操作

| 当前选择 | 可执行操作                                     |
| -------- | ---------------------------------------------- |
| 房间     | 添加家具、添加储物设备、添加物品、查看内部内容 |
| 家具     | 添加储物设备、添加物品、查看内部内容           |
| 储物设备 | 添加物品、查看内部物品                         |
| 物品     | 修改名称、数量和存放位置                       |

上下文操作可以保证用户只看到当前层级允许执行的动作，避免顶部工具栏持续增加按钮。

## 3. 房间

房间继续沿用当前已经实现的创建、选择、移动、缩放、重命名、删除、搜索和定位能力。

新增家具和储物设备后，删除房间必须先展示其内部实体数量，并把整次删除作为一条可撤销操作。

## 4. 家具创建与编辑

### 4.1 创建家具

1. 选择目标房间。
2. 在右侧详情面板点击“添加家具”。
3. 进入一次性家具绘制状态。
4. 在目标房间内部拖拽创建家具。
5. 创建完成后自动退出绘制状态并选中新家具。

家具位置和尺寸限制在所属房间范围内。

### 4.2 家具操作

- 移动和调整大小。
- 重命名和删除。
- 查看内部储物设备和物品。
- 转移到其他房间。
- 显示内部物品总数。

家具转移房间后，其内部设备和物品自动跟随，不逐项要求用户重新选择位置。

## 5. 储物设备创建与编辑

### 5.1 房间级储物设备

- 选择房间后添加。
- 在房间画布内显示矩形节点。
- 支持移动、调整大小、重命名和删除。
- 可以直接存放物品。

### 5.2 家具内储物设备

- 选择家具后添加。
- 第三阶段已作为家具详情中的逻辑子节点实现。
- 支持重命名、删除和存放物品。
- 当前不记录家具内部的精确二维坐标。

### 5.3 位置切换

- 房间级设备移入家具后，移除房间画布矩形数据。
- 家具内设备移动到房间后，在房间中心生成默认矩形。
- 设备位置变化后，内部物品路径自动更新。

## 6. 物品快速添加

当前新增物品表单包含：

- 物品名称。
- 数量，默认值为 1。
- 存放位置。

### 6.1 默认位置

- 选中储物设备：默认放入该储物设备。
- 选中家具：默认放入该家具。
- 选中房间：默认直接放入该房间。
- 没有有效选择：要求用户通过位置选择器选择房间和容器。

### 6.2 位置选择器

位置选择器按照层级逐步选择：

```text
房间 → 家具或房间级储物设备 → 最终位置
```

选择器只显示符合实体层级规则的目标位置。第一阶段不加入图片、价格、购买时间、保质期和复杂分类。

### 6.3 物品操作

- 修改名称和数量。
- 移动到其他房间、家具或储物设备。
- 删除物品。
- 查看完整位置路径。

## 7. 全局搜索

### 7.1 搜索范围

现有房间搜索升级为全局搜索，覆盖房间、家具、储物设备和物品。

第一阶段继续使用前端内存过滤，不引入独立搜索服务或搜索索引库。

### 7.2 结果展示

搜索结果同时显示实体类型、名称和完整路径：

```text
遥控器 · 物品
客厅 / 电视柜

保鲜袋 · 物品
厨房 / 橱柜 / 左侧收纳盒
```

### 7.3 点击结果

1. 定位并居中目标所属房间。
2. 选中并高亮所属家具或房间级储物设备。
3. 打开右侧详情面板。
4. 展开目标所在的容器层级。
5. 滚动并高亮最终目标。
6. 把键盘焦点交给对应画布节点或详情列表项。

## 8. 删除与级联

删除包含子内容的实体前必须显示影响数量：

```text
删除“电视柜”将同时影响：
- 2 个储物设备
- 16 件物品
```

第一阶段采用“确认后级联删除 + 支持撤销”：

- 删除房间：删除其家具、房间级储物设备和全部后代物品。
- 删除家具：删除其内部储物设备和直接或间接存放的物品。
- 删除储物设备：删除其中的物品。
- 删除物品：只删除该物品。

后续确认有需求后，再增加“删除容器并转移内部内容”。

## 9. 键盘与无障碍

- 新增的空间节点沿用当前房间的键盘选择、移动、重命名和删除方式。
- 详情面板支持方向键浏览层级和 `Escape` 关闭。
- 搜索定位后把焦点移动到最终目标。
- 对话框关闭后恢复触发元素焦点。
- 所有按钮、输入框和层级节点提供明确的无障碍名称。
- 粗指针设备继续保持至少 44 × 44 像素的主要点击区域。

## 10. 产品交互验收

- 用户只能在房间内创建家具。
- 用户只能把储物设备放到房间或家具。
- 用户只能把物品放到房间、家具或储物设备。
- 新增物品时优先使用当前选择作为默认位置。
- 移动父容器后，物品显示路径立即更新。
- 搜索结果可以定位、展开并高亮最终目标。
- 删除容器前能够看到受影响的设备和物品数量。
- 整次级联操作可以一次撤销。
