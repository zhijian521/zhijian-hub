## 组件分层规范

### 三层组件架构

| 层       | 目录                   | 定位                         | 依赖                     |
| -------- | ---------------------- | ---------------------------- | ------------------------ |
| 原子组件 | `components/ui/`       | 无业务语义、可跨项目复用     | 通用工具                 |
| 通用组件 | `components/common/`   | 有项目语义，但不绑定具体路由 | UI 组件 + 业务数据       |
| 业务组件 | `components/features/` | 由多个组件组成的业务区块     | UI + 通用组件 + 业务逻辑 |

### 判断标准

- **原子组件**：Button、Input、Dialog 等基础 UI 元素，不包含业务逻辑
- **通用组件**：Header、Footer、Card 等包含项目特色的组件
- **业务组件**：LoginForm、UserList 等特定业务功能的组件

## 文件组织规范

### 目录结构

推荐使用"一个组件一个目录"的方式:

```text
components/ui/button/
├── index.ts           # 统一导出
├── button.tsx         # 组件实现
└── button.module.css  # 样式文件
```

### 命名规则

- **目录名**: kebab-case（如 `ghost-button`）
- **文件名**: kebab-case（如 `ghost-button.tsx`）
- **导出名**: PascalCase（如 `GhostButton`）
- **样式**: CSS Modules + camelCase

## 组件实现规范

### 函数组件

```tsx
/*============================================================================
  ghost-button — 透明背景按钮组件

  提供不同主题的按钮变体，支持点击事件处理
============================================================================*/

/*== 依赖导入 ==*/
import React from 'react';

/*== 组件导入 ==*/

/*== 样式导入 ==*/
import styles from './button.module.css';

/*== 类型定义 ==*/
interface GhostButtonProps {
    /*-- 按钮内容 --*/
    children: React.ReactNode;
    /*-- 按钮主题，默认 default --*/
    variant?: 'default' | 'primary';
    /*-- 是否禁用，默认 false --*/
    disabled?: boolean;
}

/*== GhostButton 组件 — 透明背景按钮 ==*/
export function GhostButton({ children, variant = 'default', disabled = false }: GhostButtonProps) {
    return (
        <button className={styles.button} data-variant={variant} disabled={disabled}>
            {children}
        </button>
    );
}
```

### 最佳实践

1. **统一函数组件**：使用 `function` 关键字，不用箭头函数
2. **Props 类型**：使用 `interface`，就近定义
3. **默认值**：函数参数解构时设置
4. **样式优先**：使用 CSS Modules，避免内联 `style`
5. **副作用控制**：业务逻辑放在 hook 或容器组件
6. **注释规范**：文件头注释 + 分区注释 + 行内注释

## 优化清单

### 1. 文件结构与注释

- [ ] 文件头包含用途说明和关键约束
- [ ] 导入按依赖层级分组并注释
- [ ] Props 类型定义清晰，每个字段有说明
- [ ] 组件函数上方有功能描述注释
- [ ] JSX 关键区块有注释说明

### 2. 条件渲染优化

使用 `<Show>` 组件替代三元运算符：

```tsx
// ❌ 不推荐
{
    condition ? <Component /> : null;
}

// ✅ 推荐
<Show when={condition}>
    <Component />
</Show>;
```

### 3. CSS 优化

- [ ] 样式按区块分组，用 `/*-- xxx --*/` 注释
- [ ] 颜色使用 CSS 变量（如 `var(--primary)`）
- [ ] 间距使用间距变量（如 `var(--space-4)`）
- [ ] 字号使用字号变量（如 `var(--text-base)`）
- [ ] 圆角使用圆角变量（如 `var(--radius)`）

### 4. 性能优化

- [ ] 避免不必要的重新渲染
- [ ] 合理使用 React.memo
- [ ] 避免在渲染函数中创建对象
- [ ] 使用 useMemo/useCallback 优化计算

### 5. 可访问性

- [ ] 添加适当的 ARIA 属性
- [ ] 键盘导航支持
- [ ] 语义化 HTML 标签
- [ ] 适当的颜色对比度

## 提交前检查

创建或修改组件后，运行以下检查：

```bash
npm run check  # 代码检查
npm run format # 格式化代码
```

确保：

- 代码风格符合项目规范
- 类型检查通过
- 没有 ESLint 错误或警告
- 组件功能完整且可复用
