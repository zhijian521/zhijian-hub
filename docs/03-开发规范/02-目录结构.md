# 02 · 目录结构

- 文档状态：开发规范
- 适用版本：`zhijian-hub@0.1.0`
- 最后更新：2026-07-18
- 维护范围：仓库目录职责、组件分层和路径规则
- 关联代码：`src/`、`public/`、`docs/`

## 顶层

```
project-root/
├── src/                # 源代码目录
├── public/             # 静态资源
├── docs/               # 产品、技术、开发规范和方案文档
├── next.config.ts      # Next.js 配置
├── tsconfig.json       # TypeScript 配置
└── package.json        # NPM 配置
```

## src 目录结构

```
src/
├── app/                        # Next.js App Router:路由 + 页面
│   ├── api/                    # API Route
│   ├── layout.tsx              # 根布局组件
│   ├── page.tsx                # 首页
│   └── globals.css             # 全局样式重置
├── components/                 # React 组件库
│   ├── ui/                     # 原子组件:Button, Input, Dialog 等
│   ├── site/                   # 站点级品牌与布局组件
│   └── features/               # 按业务功能组织的复合组件
├── lib/                        # 工具库
│   ├── hooks/                  # 自定义 Hook
│   ├── types/                  # 类型定义
│   └── utils/                  # 工具函数
└── styles/                     # 全局样式
    ├── tokens.css              # 间距、字号和圆角令牌
    └── theme.css               # 主题定义
```

## 目录职责

### app/

只放路由、页面、layout。业务逻辑不进 app 目录，组件从 `components/` 导入。

- `app/api/`：API 路由接口
- `app/[path]/page.tsx`：动态路由页面
- `layout.tsx`：布局组件

### components/

| 目录        | 放什么   | 判定标准                                 |
| ----------- | -------- | ---------------------------------------- |
| `ui/`       | 原子组件 | Button、Input、Dialog 等可复用的 UI 元素 |
| `site/`     | 站点组件 | 品牌标识、站点导航等全站元素             |
| `features/` | 业务组件 | 按功能组织的业务区块与交互组件           |

### lib/

存放项目的基础工具库：

- `hooks/`：自定义 Hook（按功能命名，如 `use-room-canvas.ts`）
- `types/`：类型定义（按功能命名，如 `room-canvas.ts`）
- `utils/`：通用工具函数（如 `cn.ts`）

### styles/

全局样式组织：

- `tokens.css`：间距、字号、行高、字重和圆角令牌
- `theme.css`：主题相关样式
- `app/globals.css`：全局样式重置，并按顺序导入 token 和 theme

## 文件粒度

- 组件文件使用 kebab-case，导出名使用 PascalCase（如 `ghost-button.tsx` → `GhostButton`）
- 样式文件使用同名 CSS Modules（如 `ghost-button.module.css`）
- 避免创建空目录，按需添加

## 路径别名

`tsconfig.json` 配置 `@/*` 指向 `src/*`。import 统一使用别名路径。
