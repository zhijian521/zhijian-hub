## 三层分层

样式分三层，从上到下覆盖：

| 层         | 文件                            | 内容                              |
| ---------- | ------------------------------- | --------------------------------- |
| 全局 token | `tokens.css` 或 `variables.css` | spacing / text / leading / radius |
| 主题       | `theme.css`                     | 颜色、字体、阴影、过渡、焦点      |
| 模块覆盖   | 模块根组件的 CSS Module         | 仅该模块自己的变量差异            |

全局样式文件固定导入顺序：

```css
@import './tokens.css';
@import './theme.css';
```

原则：

- token 先于 theme。
- token 不放颜色。
- theme 可以引用 token。
- 模块变量只在模块根容器里声明，不污染全局 `:root`。

## token 命名以项目实际为准

实际项目中使用的 token 变量以当前仓库的真实实现为准。新增样式请参考已有 token，避免重新发明一样的变量。

### spacing

```css
--space-1: 0.25rem; /* 4px */
--space-2: 0.5rem; /* 8px */
--space-3: 0.75rem; /* 12px */
--space-4: 1rem; /* 16px */
--space-5: 1.5rem; /* 24px */
--space-6: 2rem; /* 32px */
--space-7: 3rem; /* 48px */
--space-8: 4rem; /* 64px */
```

### text

```css
--text-xs
--text-sm
--text-base
--text-lg
--text-xl
--text-2xl
--text-3xl
```

### line-height

```css
--leading-tight
--leading-normal
--leading-relaxed
```

### radius

```css
--radius-sm: 0.25rem; /* 4px：空间对象 */
--radius: 0.375rem; /* 6px：交互控件 */
--radius-lg: 0.625rem; /* 10px：浮层容器 */
```

圆角按语义分层：房间与绘制预览使用 `--radius-sm`，按钮与输入框使用 `--radius`，工具栏、菜单和弹窗等浮层使用 `--radius-lg`。画布、网格与分隔线保持直角。

## 主题语义

来自 `theme.css` 的常用语义变量包括：

- 文本：`--foreground` `--muted-foreground`
- 主色：`--primary` `--primary-foreground` `--primary-hover` `--primary-active`
- 语义色：`--success` `--warning` `--info` `--destructive`
- 背景：`--background` `--highlight` `--muted` `--secondary`
- 边框 / 输入：`--border` `--border-strong` `--input`
- 浮层：`--card` `--popover` `--overlay`
- 阴影 / 焦点 / 过渡：`--shadow-sm` `--shadow` `--shadow-lg` `--focus-outline` `--transition`

不要在 spec 里写不存在的变量名，例如 `--text-md`、`--text-primary`。当前正确写法分别是 `--text-base` 和 `--foreground`。

## 使用规则

推荐写法：

```css
.button {
    padding: var(--space-2) var(--space-4);
    font-size: var(--text-sm);
    line-height: var(--leading-normal);
    color: var(--foreground);
    background: var(--primary);
    border-radius: var(--radius);
}
```

禁止：

- `padding: 12px`
- `font-size: 14px`
- `color: #333`
- 在 `tokens.css` 里放颜色
- 为模块差异新建全局 `modules/` 样式层

## 不强行 token 化的例外

以下值属于刻度之外，保留原值，不硬塞 token：

- **像素级精度**：热力图单元格、分隔线、icon 占位的 `11px` / `3px` / `2px` 等需要像素锁定的尺寸。
- **响应式值**：`clamp()`、`100vh` / `100vw`、媒体查询断点。
- **渐变端点**：`linear-gradient(... var(--background) 2rem)` 里的 `2rem` 是渐变长度，不是间距。
- **`border-width`**：边框宽度不用 `--space-*`（间距 token 语义是间距），写裸 `1px` / `2px` 或新增 `--border-width-*`。

判定原则：token 表达的是「刻度」（间距阶梯、字号阶梯），不是「所有 rem/px」。强塞 token 反而语义错位。拿不准时，问自己「这个值属于哪个刻度」——答不上来就保留原值。

## CSS Modules

- 一个组件配一个同名 `.module.css`
- class 名统一 camelCase
- 非必要不用 `:global`
- 组合优先用 `composes` 或 class 组合，不搞继承链花活

## 模块变量

模块需要局部变量时，只在模块根容器声明：

```css
.navRoot {
    --nav-bg: var(--secondary);
    --nav-gap: var(--space-2);
    background: var(--nav-bg);
    gap: var(--nav-gap);
}
```

子组件只读取 `var(--nav-bg)`，不要回写全局变量。

## 现有硬编码怎么处理

清理硬编码是日常维护的一部分，不要求一次性全仓扫完。规则是：

- 新写组件必须直接用 token / theme 变量
- 改到旧组件时，顺手把碰到的硬编码替掉
- 不为了“纯洁度”做一轮只改样式值、不改功能的大面积调整
